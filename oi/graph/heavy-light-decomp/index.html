图论 - 树链剖分

发表于2023-10-17|更新于2023-11-04|OI
|字数总计:12.1k|阅读时长:61分钟

简介

树链剖分通常是用来解决对树上路径进行修改查询的这一类问题，它可以把树上路径转化为 O(log⁡n)O(\log n)O(logn) 段序列的连续区间，通常使用线段树维护这些区间，所以单次询问的复杂度会是 O(log⁡2n)O(\log^2 n)O(log2n)。

接下来介绍概念，对于结点 uuu 和它的子结点 vvv：

  • 如果 sz(v)sz(v)sz(v) 是所有子结点中最大的，称 vvv 为重儿子，否则为轻儿子。若有多个相同任选一个即可。
  • 若 vvv 为重儿子，那么边 (u,v)(u,v)(u,v) 为重边，否则为轻边。

我们对于每个结点 uuu 都记录一个 top(u)top(u)top(u)，代表 uuu 所在重链中深度最小的那个结点，所以我们可以以 O(1)O(1)O(1) 的时间从重链上任意一个结点跳转到链顶。

接下来简单证明一下为什么可以分成 O(log⁡n)O(\log n)O(logn) 条连续路径：

  • 如果 (u,v)(u,v)(u,v) 为重边，可以由上面说的那样以 O(1)O(1)O(1) 的时间跳转到链顶。
  • 如果 (u,v)(u,v)(u,v) 为轻边，即 vvv 为轻儿子，那么一定有 sz(v)≤12∑isz(vi)sz(v)\le \frac{1}{2}\sum_i sz(v_i)sz(v)≤21​∑i​sz(vi​)，否则 vvv 就一定是重儿子了。那么每次经过一条轻边，点的数量都会减少一半，所以会经过 O(log⁡n)O(\log n)O(logn) 条轻边，将这条路径分割成了 O(log⁡n)O(\log n)O(logn) 条连续区间，注意一条重链上的区间是连续的。

我们进行树剖时，会给每个点用 dfsdfsdfs 序重新标号，每次优先走重儿子，所以这样整条重链上的标号都是连续的，也就可以用线段树进行区间操作。

[模板] 树链剖分

给定一棵 nnn 个节点的树，初始时该树的根为 111 号节点，每个节点有一个给定的权值。下面依次进行 mmm 个操作，操作分为如下五种类型：

  • 换根：将一个指定的节点设置为树的新根。
  • 修改路径权值：给定两个节点，将这两个节点间路径上的所有节点权值（含这两个节点）增加一个给定的值。
  • 修改子树权值：给定一个节点，将以该节点为根的子树内的所有节点权值增加一个给定的值。
  • 询问路径：询问某条路径上节点的权值和。
  • 询问子树：询问某个子树内节点的权值和。

输入格式

第一行一个整数 nnn，表示节点的个数。

第二行 nnn 个整数表示第 iii 个节点的初始权值 wiw_iwi​。

第三行 n−1n-1n−1 个整数，表示 i+1i+1i+1 号节点的父节点编号 fi+1 (1≤fi+1≤n)f_{i+1}\ (1 \le f_{i+1} \le n)fi+1​ (1≤fi+1​≤n)。

第四行一个整数 mmm，表示操作个数。

接下来 mmm 行，每行第一个整数表示操作类型编号：(1≤u,v≤n)(1 \le u, v \le n)(1≤u,v≤n)

  • 若类型为 111，则接下来一个整数 uuu，表示新根的编号。
  • 若类型为 222，则接下来三个整数 u,v,ku,v,ku,v,k，分别表示路径两端的节点编号以及增加的权值。
  • 若类型为 333，则接下来两个整数 u,ku,ku,k，分别表示子树根节点编号以及增加的权值。
  • 若类型为 444，则接下来两个整数 u,vu,vu,v，表示路径两端的节点编号。
  • 若类型为 555，则接下来一个整数 uuu，表示子树根节点编号。

我们使用两个 dfsdfsdfs 预处理树上信息，第一个处理出每个点的深度 depdepdep，子树大小 szszsz，重儿子 sonsonson；第二个处理 dfsdfsdfs 序 dfndfndfn，重链头标号 toptoptop。

每次处理 u,vu,vu,v 路径上的点时，每次挑 top(u),top(v)top(u),top(v)top(u),top(v) 较深的那个跳上去，直到它们跳到同一条重链上时最后一段路径就是 [dfn(u),dfn(v)][dfn(u),dfn(v)][dfn(u),dfn(v)] 之间的区间。

对于 uuu 的子树，区间为 [dfn(u),dfn(u)+sz(u)−1][dfn(u),dfn(u)+sz(u)-1][dfn(u),dfn(u)+sz(u)−1]，这个在前面的树形背包中也有所涉及。

对于换根操作，路径是不关心根节点是哪里的，只有询问子树的时候才需要关心根节点是谁：

  • 若 u=rtu=rtu=rt，那么操作的是整棵树。
  • 若 LCA(u,rt)=uLCA(u,rt)=uLCA(u,rt)=u 且 u≠rtu\ne rtu=rt，那么操作整棵树后撤销对于 rt→urt\to urt→u 路径上的倒数第二个点的子树的操作。
  • 若 LCA(u,rt)≠uLCA(u,rt)\ne uLCA(u,rt)=u，那么还是 uuu 的子树不变。

这与前文提到的换根 LCA 操作比较类似。

  2. 传递情报：将一条情报从 XXX 号情报员传递给 YYY 号情报员。

情报员最初处于潜伏阶段，他们是相对安全的，我们认为此时所有情报员的危险值为 000；一旦某个情报员开始搜集情报，他的危险值就会持续增加，每天增加 111 点危险值 (开始搜集情报的当天危险值仍为 000，第 222 天危险值为 111，第 333 天危险值为 222，以此类推)。传递情报并不会使情报员的危险值增加。

为了保证传递情报的过程相对安全，每条情报都有一个风险控制值 CCC。奈特公司认为，参与传递这条情报的所有情报员中，危险值大于 CCC 的情报员将对该条情报构成威胁。现在，奈特公司希望知道，对于每个传递情报任务，参与传递的情报员有多少个，其中对该条情报构成威胁的情报员有多少个。

输入格式

第一行包含一个正整数 nnn，表示情报员个数。
笫二行包含 nnn 个非负整数，其中第 iii 个整数 PiP_iPi​ 表示 iii 号情报员上线的编号。特别地，若 Pi=0P_i=0Pi​=0，表示 iii 号情报员是大头目。
第三行包含一个正整数 qqq，表示奈特公司将派发 qqq 个任务 (每天一个)。

随后 qqq 行，依次描述 qqq 个任务。每行首先有一个正整数 kkk。

  • 若 k=1k=1k=1，表示任务是传递情报，随后有三个正整数 XiX_iXi​、YiY_iYi​、CiC_iCi​，依次表示传递情报的起点、终点和风险控制值。
  • 若 k=2k=2k=2，表示任务是搜集情报，随后有 111 个正整数 TiT_iTi​，表示搜集情报的情报员编号。

输出格式

对于每个传递情报任务输出一行，包含两个整数，分别是参与传递情报的情报员个数和对该条情报构成威胁的情报员个数。

数据范围：n⩽2×105,Q⩽2×105,0<Pi,Ci⩽N,1⩽Ti,Xi,Yi⩽nn\leqslant 2\times 10^5,Q\leqslant 2\times 10^5,0<P_i,C_i\leqslant N,1\leqslant T_i,X_i,Y_i\leqslant nn⩽2×105,Q⩽2×105,0<Pi​,Ci​⩽N,1⩽Ti​,Xi​,Yi​⩽n。

题目链接：P4216。

操作抽象一下，就是这样：

  1. 查询 X,YX,YX,Y 路径上满足 T−Tu>CT-T_u>CT−Tu​>C 的点的数量。
  2. 给结点打上当前时间戳。

注意到第二个操作的时间戳是严格单调递增的，我们把第一个操作移项一下：T−C>TuT-C>T_uT−C>Tu​，那么我们可以离线下来按照 T−CT-CT−C 排序，每次把满足要求的 uuu 的点权设为 111，然后求一个 X∼YX\sim YX∼Y 路径上的点权和。

对于结点个数，这个不可能不会吧，dep(X)+dep(Y)−2dep(LCA)+1dep(X)+dep(Y)-2dep(LCA)+1dep(X)+dep(Y)−2dep(LCA)+1。

由于只有单点加和区间查询，这里用树状数组比较好。
