康托展开

给定一个排列，求其按照字典序的排名，对 998244353998244353998244353 取模，N≤1000000N\le 1000000N≤1000000。

题目链接：P5367。

对于一个排列 ppp，当枚举到 pip_ipi​ 时，显然只要满足 ∀j<i,x≠pj\forall j<i,x\ne p_j∀j<i,x=pj​ 并且 x<pix\lt p_ix<pi​ 就可以作为它前驱序列的开头，然后后面做一个全排列，即 cnt(x)×(n−i)!\text{cnt}(x)\times (n-i)!cnt(x)×(n−i)! 这里 cnt(x)\text{cnt}(x)cnt(x) 可以用树状数组求。

交换次数

给定排列，求将第一个排列按照字典序的顺序转化到这个排列需要经过多少次，并且求出每次经过的步数（相邻两项交换一次算一步）。

打表

赛时遇到找规律，直接打表。赛后我竟然半天没写出来打表的程序，然后原始文件还被我覆盖掉了，太悲伤了。

首先我们将一个排列变成另一个排列，需要的最少步数，一定是先确定好第一项，然后确定第二项，依次往下，这样就能保证是最小的。

通过更改 nnn 的值造成的影响可以列出来，这里循环节指的是包含最大数字的循环节：

nnn 最大数字 循环节长度 总长
222 111 111 111
333 222 222 555
444 444 666 232323
555 777 242424 119119119
666 111111 120120120 719719719
777 161616 720720720 503950395039

可以人为的把总长加一，然后明显观察到这是阶乘的关系，然后最大数字的差分是 1,2,3,4,51,2,3,4,51,2,3,4,5 递增的关系，因此我们进行求和，可以认为对于长度为 nnn 的排列，一开始总长为 n!n!n! 的数列中全是 111：

  • 隔 2!2!2! 数字加 111
  • 隔 3!3!3! 数字加 222
  • 隔 4!4!4! 数字加 333
  • 隔 5!5!5! 数字加 444

因此：

f(n)=n!+n!2!×1+n!3!×2+⋯=n!+n!∑i=2ni−1i!\begin{aligned} f(n)&=n!+\frac{n!}{2!}\times 1+\frac{n!}{3!}\times 2+\cdots\\ &=n!+n!\sum_{i=2}^{n} \frac{i-1}{i!} \end{aligned} f(n)​=n!+2!n!​×1+3!n!​×2+⋯=n!+n!i=2∑n​i!i−1​​

我们最后把最大的那个数减掉，是 1+n(n−1)21+\frac{n(n-1)}{2}1+2n(n−1)​。

现在我们可以计算出长度为 nnn 的排列从 1∼n1\sim n1∼n 到 n∼1n\sim 1n∼1 需要的操作次数了，考虑按照康托展开那样计算。

考虑到我们上面是按照 nnn 个循环节来计算的，所以我们这里有长度为 n−i+1n-i+1n−i+1 的 cnt(x)\text{cnt}(x)cnt(x) 个循环节，猜想直接累加 cnt(x)×f(n−i+1)/(n−i+1)\text{cnt}(x)\times f(n-i+1)/(n-i+1)cnt(x)×f(n−i+1)/(n−i+1)，发现答案正确。

推公式

打表的方法看上去比较扯淡，但是考场上做出来大于一切，下面开始正经推导。

我们将排列 1∼n1\sim n1∼n 变成 n∼1n\sim 1n∼1，计作 f(n)f(n)f(n)，那么：

  1. 把 1∼n1\sim n1∼n 变成 1,n,…,21,n,\dots,21,n,…,2 需要 f(n−1)f(n-1)f(n−1)
  2. 把 222 移到头上变成 2,1,n,…,32,1,n,\dots,32,1,n,…,3 需要 n−1n-1n−1
  3. 由于 222 开头的字典序最小的是 2,1,3,…,n2,1,3,\dots,n2,1,3,…,n 翻转 n−2n-2n−2 个数字需要 ∑i=1n−3i=(n−2)(n−3)/2=Cn−22\sum_{i=1}^{n-3} i=(n-2)(n-3)/2=C_{n-2}^2∑i=1n−3​i=(n−2)(n−3)/2=Cn−22​。

所以换一个头需要的操作步数是 n−1+(n−2)(n−3)/2n-1+(n-2)(n-3)/2n−1+(n−2)(n−3)/2，我们需要 nnn 次翻转和 n−1n-1n−1 次换头，所以公式就是：

f(n)=nf(n−1)+(n−1)(n−1+Cn−22)f(n)=nf(n-1)+(n-1)(n-1+C_{n-2}^2) f(n)=nf(n−1)+(n−1)(n−1+Cn−22​)

显然 f(1)=0f(1)=0f(1)=0，递推即可。但是类似数位 DP 的感觉去思考，需要最后再来一次换头的操作，但是这里我们调用的是 f(n−i)f(n-i)f(n−i) 但是换头加的是 n−i+Cn−i−12n-i+C_{n-i-1}^2n−i+Cn−i−12​，因为是把后面整个 n−in-in−i 都换个头。
