数据结构 - Splay

发表于2023-08-06|更新于2023-12-20|OI
|字数总计:7k|阅读时长:37分钟

简介

借助左旋和右旋操作，在保证 BST 性质的情况下尽可能地减少树的高度。

当结点 x→y→zx \to y \to zx→y→z 呈直线形的时候，rotate(y),rotate(x)rotate(y), rotate(x)rotate(y),rotate(x)；呈折线形时，rotate(x),rotate(x)rotate(x),rotate(x)rotate(x),rotate(x)。可以五个结点分情况看一下，这样能保证树的高度尽可能小。

势能法摊还分析

原理

在难以求出一个数据结构真实的复杂度时，可以使用势能的概念辅助进行计算，设 cic_ici​ 是每次的真实复杂度，ci^\hat{c_i}ci​^​ 是均摊复杂度，那么有：

ci^=ci+Δφi\hat{c_i}=c_i+\Delta\varphi_i ci​^​=ci​+Δφi​

由此可以推出：

∑ci^=∑ci+φn−φ0\sum \hat{c_i}=\sum c_i+\varphi_n-\varphi_0 ∑ci​^​=∑ci​+φn​−φ0​

我们只需要找到一个合适的势能函数，就可以用 ∑ci^\sum \hat{c_i}∑ci​^​ 当作 ∑ci\sum c_i∑ci​ 的一个上界，从而进行复杂度的证明。

势能函数

为了行文方便，用 vvv 表示某个节点，TTT 表示全树，∣v∣|v|∣v∣ 表示结点对应的子树大小，定义势能函数如下：

φi=∑v∈Tlog⁡∣v∣\varphi_i=\sum_{v\in T} \log|v| φi​=v∈T∑​log∣v∣

实际意义是 log⁡∣v∣\log|v|log∣v∣ 表示 vvv 子树的期望树高。

单旋

设 x,y,zx,y,zx,y,z 后者为前者的父结点，对 xxx 进行单旋的时候，有：

Δφi=log⁡∣x′∣+log⁡∣y′∣−log⁡∣x∣−log⁡∣y∣\Delta\varphi_i=\log|x'|+\log|y'|-\log|x|-\log|y| Δφi​=log∣x′∣+log∣y′∣−log∣x∣−log∣y∣

显然有 ∣x′∣=∣y∣|x'|=|y|∣x′∣=∣y∣ 且 ∣x′∣≥∣y′∣|x'|\ge|y'|∣x′∣≥∣y′∣ 因此可以得到：

Δφi=log⁡∣y′∣−log⁡∣x∣≤log⁡∣x′∣−log⁡∣x∣\Delta\varphi_i=\log|y'|-\log|x|\le \log|x'|-\log|x| Δφi​=log∣y′∣−log∣x∣≤log∣x′∣−log∣x∣

这里有：

ci^≤1+3(log⁡∣x′∣−log⁡∣x∣)\hat{c_i}\le 1+3(\log|x'|-\log|x|) ci​^​≤1+3(log∣x′∣−log∣x∣)

这里的 333 主要是为了和下边对应起来。

双旋

同上，设 x,y,zx,y,zx,y,z 三个结点，对 xxx 单旋两次，可以得到：

Δφi=log⁡∣x′∣+log⁡∣y′∣+log⁡∣z′∣−log⁡∣x∣−log⁡∣y∣−log⁡∣z∣\Delta\varphi_i=\log|x'|+\log|y'|+\log|z'|-\log|x|-\log|y|-\log|z| Δφi​=log∣x′∣+log∣y′∣+log∣z′∣−log∣x∣−log∣y∣−log∣z∣

有 ∣x′∣=∣z∣,∣y∣≥∣x∣,∣y′∣≤∣x′∣|x'|=|z|,|y|\ge|x|,|y'|\le|x'|∣x′∣=∣z∣,∣y∣≥∣x∣,∣y′∣≤∣x′∣，那么：

Δφi≤log⁡∣x′∣+log⁡∣z′∣−2log⁡∣x∣\Delta\varphi_i\le\log|x'|+\log|z'|-2\log|x| Δφi​≤log∣x′∣+log∣z′∣−2log∣x∣

由于旋转两次，均摊复杂度为：

ci^=2+Δφi\hat{c_i}=2+\Delta\varphi_i ci​^​=2+Δφi​

有一个神秘的操作可以把常数 222 消掉，如下：

log⁡∣x∣+log⁡∣z′∣−2log⁡∣x′∣=log⁡∣x∣∣z′∣∣x′∣2≤log⁡∣x∣∣z′∣(∣x∣+∣z′∣)2≤−2\begin{aligned} \log|x|+\log|z'|-2\log|x'|&=\log\frac{|x||z'|}{|x'|^2}\\ &\le\log\frac{|x||z'|}{(|x|+|z'|)^2}\\ &\le-2 \end{aligned} log∣x∣+log∣z′∣−2log∣x′∣​=log∣x′∣2∣x∣∣z′∣​≤log(∣x∣+∣z′∣)2∣x∣∣z′∣​≤−2​

我们移项消掉 log⁡∣z′∣\log|z'|log∣z′∣ 可以得到：

Δφi≤log⁡∣x′∣+(−2+2log⁡∣x′∣−log⁡∣x∣)−2log⁡∣x∣≤−2+3(log⁡∣x′∣−log⁡∣x∣)\Delta\varphi_i\le\log|x'|+(-2+2\log|x'|-\log|x|)-2\log|x|\le-2+3(\log|x'|-\log|x|) Δφi​≤log∣x′∣+(−2+2log∣x′∣−log∣x∣)−2log∣x∣≤−2+3(log∣x′∣−log∣x∣)

可得：

ci^≤3(log⁡∣x′∣−log⁡∣x∣)\hat{c_i}\le3(\log|x'|-\log|x|) ci​^​≤3(log∣x′∣−log∣x∣)

综上

进行 nnn 次 splay 操作需要的总复杂度为：

∑ci^≤3n(log⁡∣xn∣−log⁡∣x∣)+1\sum\hat{c_i}\le 3n(\log|x_n|-\log|x|)+1 ∑ci​^​≤3n(log∣xn​∣−log∣x∣)+1

后面的 111 表示最后一次单旋，于是可以得到复杂度为 O(nlog⁡n)O(n\log n)O(nlogn)。

文艺平衡树

维护一个 nnn 项有序数列，序列中第 iii 项初始为 iii，输入 mmm 行 l,rl,rl,r 表示把区间 [l,r][l,r][l,r] 翻转。

输出进行 mmm 次变换后的序列。

题目链接：AcWing 2437，P3391。

左旋和右旋可以合并到一个函数中写，在现在的 Splay 代码中大量存在着用布尔表达式判断是哪个子结点的情况，这样可以少写很多 if 语句。

普通平衡树

您需要写一种数据结构（可参考题目标题），来维护一些数，其中需要提供以下操作：

  1. 插入 x 数；
  2. 删除 x 数（若有多个相同的数，只删除一个）；
  3. 查询 x 数的排名（若有多个相同的数，输出最小的排名）；
  4. 查询排名为 x 的数；
  5. 求 x 的前驱（前驱定义为小于 x，且最大的数）；
  6. 求 x 的后继（后继定义为大于 x，且最小的数）。

题目链接：AcWing 253，P3369。

这题比上面的文艺平衡树反而更难写，删除一个数实在是很困难的。

[NOI2004] 郁闷的出纳员

题干过长，这里不粘过来了。

输入格式

第一行有两个非负整数 n 和 min，n 表示下面有多少条命令，min 表示工资下界。

接下来的 n 行，每行表示一条命令，命令可以是以下四种之一：

  1. I 命令，格式为 I_k，表示新建一个工资档案，初始工资为 k。如果某员工的初始工资低于工资下界，他将立刻离开公司。
  2. A 命令，格式为 A_k，表示把每位员工的工资加上 k。
  3. S 命令，格式为 S_k，表示把每位员工的工资扣除 k。
  4. F 命令，格式为 F_k，表示查询第 k 多的工资。

_（下划线）表示一个空格，I 命令、A 命令、S 命令中的 k 是一个非负整数，F 命令中的 k 是一个正整数。

在初始时，可以认为公司里一个员工也没有。

输出格式

输出行数为 F 命令的条数加一。对于每条 F 命令，你的程序要输出一行，仅包含一个整数，为当前工资第 k 多的员工所拿的工资数，如果 k 大于目前员工的数目，则输出 −1。

输出文件的最后一行包含一个整数，为离开公司的员工的总数。

注意，如果某个员工的初始工资低于最低工资标准，那么将不计入最后的答案内。

题目链接：AcWing 950，P1486。

本题不改变相对大小关系，因此可以让 Splay 保序。

[HNOI2012] 永无乡

永无乡包含 n 座岛，编号从 1 到 n ，每座岛都有自己的独一无二的重要度，按照重要度可以将这 n 座岛排名，名次用 1 到 n 来表示。

某些岛之间由巨大的桥连接，通过桥可以从一个岛到达另一个岛。

如果从岛 a 出发经过若干座（含 0 座）桥可以到达岛 b，则称岛 a 和岛 b 是连通的。

现在有两种操作：

  • B x y 表示在岛 x 与岛 y 之间修建一座新桥。
  • Q x k 表示询问当前与岛 x 连通的所有岛中第 k 重要的是哪座岛，即所有与岛 x 连通的岛中重要度排名第 k 小的岛是哪座，请你输出那个岛的编号，如果不存在输出 -1。

题目链接：AcWing 1063，P3224。

首先应该开多少数组大小，最坏情况下需要把所有岛都合并，合并时把小的合并到大的里面去，这样能尽可能避免浪费空间，于是我们注意到在最坏的情况下会将两个长度为 n/2n/2n/2 的连通块合并，而得到这两个连通块的方式最坏情况下也是 n/4n/4n/4 的连通块合并，这样递归下去。

可以写出下面的函数帮助我们得到一个需要的空间：

f(x)={32x+f(12x),x>10,x≤1f(x)=\left\{\begin{aligned} &\frac{3}{2}x+f(\frac{1}{2}x),x>1\\ &0,x\le1 \end{aligned}\right. f(x)=⎩⎨⎧​​23​x+f(21​x),x>10,x≤1​

用 matplotlib 画了一下，惊奇地发现这是一条直线，于是继续展开，然后小小放缩了一下：

f(x)=32x+32(12x)+⋯+32(12nx)=32x(1+12+⋯+12n)=3x(1−12n+1)<3x\begin{aligned} f(x)&=\frac{3}{2}x+\frac{3}{2}(\frac{1}{2}x)+\cdots+\frac{3}{2}(\frac{1}{2^n}x)\\ &=\frac{3}{2}x(1+\frac{1}{2}+\cdots+\frac{1}{2^n})\\ &=3x(1-\frac{1}{2^{n+1}})\\ &\lt 3x \end{aligned} f(x)​=23​x+23​(21​x)+⋯+23​(2n1​x)=23​x(1+21​+⋯+2n1​)=3x(1−2n+11​)<3x​

所以这可以近似的认为是直线 y=3xy=3xy=3x，根据本题数据，保险点开 350000 就够了。
