数据结构 - Hash

发表于2023-06-15|更新于2023-08-09|OI
|字数总计:1.5k|阅读时长:6分钟

简介

散列函数将大范围的数字映射为小范围的数字，但不保证单调性，即多个值可能映射为同一个值，这种情况叫做哈希碰撞。对于整数 MMM 来说，一般求 hash 值直接取模 M%NM\%NM%N 作为映射到 [0,N−1][0, N-1][0,N−1] 的 hash 值，这里的 NNN 最好是一个质数才保证碰撞发生概率小，证明如下（M,NM, NM,N 均为正数且 m,nm,nm,n 为它们的约数）：

M=qN+r,m=qn+rk⇒m%n=rk∈{0,1,2,⋯ ,m−1}⇒r∈{0,k,2k,⋯ ,k(m−1)}\begin{aligned} M&=qN+r,m=qn+\frac{r}{k}\\ \Rightarrow m\%n&=\frac{r}{k}\in\{0,1,2,\cdots,m-1\}\\ \Rightarrow r&\in\{0,k,2k,\cdots,k(m-1)\} \end{aligned} M⇒m%n⇒r​=qN+r,m=qn+kr​=kr​∈{0,1,2,⋯,m−1}∈{0,k,2k,⋯,k(m−1)}​

因此，要使得 M%NM\%NM%N 能尽可能取到多的数字，让 NNN 是质数是最好的选择，这样可以保证最小公因数为 111。

字符串哈希

给定一个长度为 n∈[1,105]n \in [1, 10^5]n∈[1,105] 的字符串，再给定 m∈[1,105]m \in [1, 10^5]m∈[1,105] 个询问，每个询问包含四个整数 l1,r1,l2,r2l1,r1,l2,r2l1,r1,l2,r2，请你判断 [l1,r1][l1,r1][l1,r1] 和 [l2,r2][l2,r2][l2,r2] 这两个区间所包含的字符串子串是否完全相同。字符串中只包含大小写英文字母和数字。

可快速判断字符串是否相等，原理是将字符串看作一个 PPP 进制的数字，然后对 QQQ 取模求哈希值。一般来说，P=131,13331,Q=264P=131, 13331, Q=2^{64}P=131,13331,Q=264，这样取的原因是 131 是 128 后第一个质数，13331 也是质数；QQQ 取这个数的原因是 unsigned long long 自然溢出就等同于对 2642^{64}264 取模。

在计算时，仿照前缀和的形式，将字符串从第 111 位到第 iii 位的子区间求哈希，方法是左边为高位，右边为低位，每次将上一次求的结果乘 PPP 加上当前位置的数值就是所要得到的结果。

hashrange 函数的原理是，将 [1,l−1][1, l-1][1,l−1] 与 [1,r][1, r][1,r] 区间的值对齐后，让 [1,r][1, r][1,r] 把这个前缀减掉，就是我们要求得的区间内哈希值。由于左边是高位，右边是低位，所以 [1,l−1][1, l-1][1,l−1] 这个区间比另一个区间要少的位数，就是 [l,r][l, r][l,r] 这个区间的长度，即 r−l+1r-l+1r−l+1，因此计算式是 h[r] - h[l-1] * p[r-l+1]

注意，这里一定不要用 pow 函数求 p 的几次方，会溢出（特殊处理后应该是可以用的，但是这里就不多介绍了）。

理论上来说，这是有可能发生哈希碰撞的，但概率极低，一般可忽略不计。
